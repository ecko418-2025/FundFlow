# 系统身份与权限说明

本文档用于说明“贷管家”系统中不同登录身份可访问的页面、可执行的业务操作，以及审批流中的权限边界。

当前系统包含三类身份：

- Admin：系统管理员 / 审核人
- Operator：财务经办员 / 制单人
- LP：出资人 / 投资人查看端

## 1. 总体权限原则

系统采用“经办 - 审核 - 查看”的分权模式。

Admin 拥有最高业务权限，负责系统配置、数据维护、流水审核、收益分配审核和安全日志查看。

Operator 负责日常制单，可以录入资金流水、项目收支和收益分配方案，但其提交的数据默认进入待审核状态，不直接影响正式财务结果。

LP 只能查看与自己绑定的出资人账户相关的已审核数据，不能录入、修改、删除或审批任何业务数据。

## 2. 身份与入口

| 身份 | 登录后默认入口 | 角色字段 |
|---|---|---|
| Admin 管理员 | `/admin` | `admin` |
| Operator 经办员 | `/admin` | `operator` |
| LP 出资人 | `/lp` | `lp` |

如果用户访问了自己无权访问的路由，系统会自动重定向到该身份对应的首页。

## 3. Admin 管理员权限

### 可访问页面

| 页面 | 路由 | 权限说明 |
|---|---|---|
| 总览数据 | `/admin` | 查看系统整体经营和资金概览 |
| 资金池管理 | `/admin/pools` | 查看、新建、编辑资金池 |
| 资金池详情 | `/admin/pools/:id` | 查看资金池详情、成员、项目、流水、分配记录 |
| 出资方管理 | `/admin/investors` | 查看、新建、编辑 LP / 出资方 |
| 出资方详情 | `/admin/investors/:id` | 查看单个出资方的资金池、项目、流水、收益分配 |
| 项目管理 | `/admin/projects` | 查看、新建、编辑项目 |
| 项目详情 | `/admin/projects/:id` | 查看项目详情、维护项目出资方、登记项目收支 |
| 核心流水账 | `/admin/transactions` | 录入、导入、导出、审核、驳回、删除流水 |
| 收益分配 | `/admin/distribution` | 计算、确认、审核、驳回、删除收益分配 |
| 操作安全日志 | `/admin/audit-logs` | 查看系统操作日志 |
| 系统设置 | `/admin/settings` | 管理操作员、系统标签、全局数据校准 |
| 财务报表中心 | `/admin/reports` | 查看和导出报表 |

### 可执行业务动作

Admin 可以直接创建已生效的业务数据，包括：

- 新建资金池
- 编辑资金池
- 维护资金池出资方
- 新建 LP / 出资方
- 编辑 LP / 出资方
- 新建项目
- 编辑项目
- 维护项目出资方
- 录入资金流水
- 导入资金流水
- 创建收益分配方案
- 删除收益分配方案
- 下载模板、导出 Excel、打印 PDF
- 执行全局财务数据校准
- 新建或移除 Operator 角色映射

Admin 也是唯一可以执行审核动作的身份：

- 审核通过待审核流水
- 驳回待审核流水
- 审核通过待审核收益分配
- 驳回待审核收益分配

### 审批影响

Admin 创建的资金流水默认状态为 `approved`，会立即影响：

- 资金池可用余额
- 项目累计投资额
- 项目累计回款额
- 项目出资方实缴额
- 资金池成员累计实缴额
- 资金池成员动态份额

Admin 创建的收益分配默认状态为 `confirmed`，会进入正式收益分配记录。

## 4. Operator 经办员权限

### 可访问页面

| 页面 | 路由 | 权限说明 |
|---|---|---|
| 总览数据 | `/admin` | 查看管理端总览 |
| 资金池管理 | `/admin/pools` | 查看资金池列表和详情 |
| 资金池详情 | `/admin/pools/:id` | 查看资金池信息 |
| 项目管理 | `/admin/projects` | 查看项目列表 |
| 项目详情 | `/admin/projects/:id` | 查看项目详情，并可登记项目收支 |
| 核心流水账 | `/admin/transactions` | 录入和导入流水，提交后等待审核 |
| 收益分配 | `/admin/distribution` | 创建收益分配方案，提交后等待审核 |

### 不可访问页面

Operator 不能访问以下管理员专属页面：

- 出资方管理
- 出资方详情
- 操作安全日志
- 系统设置
- 财务报表中心

### 可执行业务动作

Operator 可以执行制单类动作：

- 录入资金流水
- 导入资金流水
- 登记项目收支
- 创建收益分配方案
- 查看资金池、项目、流水、收益分配相关页面
- 删除自己创建且仍处于待审核状态的流水

### 不可执行业务动作

Operator 不能执行审核类或系统管理类动作：

- 不能审核通过流水
- 不能驳回流水
- 不能审核收益分配
- 不能驳回收益分配
- 不能查看操作安全日志
- 不能管理系统设置
- 不能新建或移除操作员
- 不能直接创建已生效流水
- 不能直接创建已确认收益分配

### 审批影响

Operator 提交的资金流水默认状态为 `pending`。

待审核流水不会影响：

- 资金池可用余额
- 项目累计投资额
- 项目累计回款额
- 项目出资方实缴额
- 资金池成员累计实缴额
- 资金池成员动态份额

只有 Admin 审核通过后，流水状态变为 `approved`，系统才会触发正式财务重算。

Operator 提交的收益分配默认状态为 `pending`。

只有 Admin 审核通过后，收益分配状态才会变为 `confirmed`。

## 5. LP 出资人权限

### 可访问页面

| 页面 | 路由 | 权限说明 |
|---|---|---|
| 我的资产总览 | `/lp` | 查看本人相关资产、收益、项目、资金池汇总 |
| 对账账单 | `/lp/statement` | 查看本人已审核资金流水对账单 |

### 可查看数据

LP 只能查看与自己绑定的 `investor_id` 相关的数据。

主要包括：

- 本人参与的资金池汇总
- 本人参与的项目汇总
- 本人收益分配汇总
- 本人已审核资金流水
- 本人对账单打印 / PDF 导出

### 数据过滤规则

LP 端只展示已生效数据：

- 资金流水只统计 `status = 'approved'`
- 待审核流水不会展示为正式财务结果
- 待审核收益分配不会作为正式收益记录

### 不可执行操作

LP 不能执行任何后台管理动作：

- 不能创建资金池
- 不能创建项目
- 不能录入流水
- 不能导入数据
- 不能编辑 LP 信息
- 不能审核或驳回数据
- 不能查看操作安全日志
- 不能访问系统设置

## 6. Maker-Checker 审批规则

系统中的核心审批机制为 Maker-Checker：

| 动作 | Admin | Operator | LP |
|---|---|---|---|
| 创建资金流水 | 直接生效 | 待审核 | 不允许 |
| 导入资金流水 | 直接生效 | 待审核 | 不允许 |
| 登记项目收支 | 直接生效 | 待审核 | 不允许 |
| 创建收益分配 | 直接确认 | 待审核 | 不允许 |
| 审核通过流水 | 允许 | 不允许 | 不允许 |
| 驳回流水 | 允许 | 不允许 | 不允许 |
| 审核通过收益分配 | 允许 | 不允许 | 不允许 |
| 驳回收益分配 | 允许 | 不允许 | 不允许 |
| 查看安全日志 | 允许 | 不允许 | 不允许 |

## 7. 资金池互转规则

资金池与资金池之间转款会生成一组双分录：

- 出款池：生成一笔流出流水
- 入款池：生成一笔流入流水

这两笔流水视为同一组业务动作。

处理规则：

- 审核通过任意一笔，同组另一笔同步通过
- 驳回任意一笔，同组另一笔同步驳回
- 删除任意一笔，同组另一笔同步删除

这样可以避免只处理单边流水导致数据库金额关系不平衡。

## 8. 操作安全日志权限

操作安全日志只允许 Admin 查看。

日志记录范围包括：

- 登录 / 退出
- 资金池创建和编辑
- 出资方创建和编辑
- 项目创建和编辑
- 资金流水提交、审批、驳回、删除
- 收益分配提交、审批、驳回、删除
- 导入、导出、下载模板、打印 PDF
- 系统标签配置变更
- 操作员账号映射新增和删除
- 全局财务数据校准

日志时间统一按北京时间写入。

业务页面不提供删除或修改安全日志的入口。

## 9. 权限边界说明

当前权限控制主要分为两层：

第一层是路由和菜单权限。不同身份登录后，只能看到和访问对应角色允许的页面。

第二层是业务动作权限。即使页面可访问，涉及审核、确认、删除等关键动作时，底层逻辑也会再次判断角色。

因此，Operator 即使进入管理端页面，也不能绕过审批直接写入正式财务结果。

LP 即使知道管理端路由地址，也会被系统重定向回 LP 页面。
